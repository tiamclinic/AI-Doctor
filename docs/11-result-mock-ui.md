## 概要

依頼者提供の**診断結果モック**を基準に、結果画面の情報階層とビジュアルを寄せる。§4.9 の (1) 総合評価ヒーロー、(2) パーツ分析セクション、(3) 総評ブロックの**レイアウト枠**を実装する（ドクター記述の実データは T-13〜T-15 で接続）。

## ゴール / 受け入れ基準

- [ ] **総合評価ブロック**: TIAM バランス指数を主役に、短い要約文（AI `overallComment` の抜粋 or 固定コピー）と **6 大指標の横バー**（またはモックに近いメーター）を同一ブロック内に配置
- [ ] **パーツ分析**: 目・鼻・口元・輪郭・左右対称性など、**カードグリッド**でセクション化（初期はプレースホルダ or AI スコア連動の短文でも可。最終文言は T-12 / 院方データと整合）
- [ ] **総評エリア**: 既存 `DiagnosisText` をモックの「読み物」位置に移動・再スタイル（上記ヒーローと役割分担が分かる見出し）
- [ ] スマホ優先の縦スクロール・既存機能（理想顔・シェア・SNS）を破壊しない
- [ ] デザイントークン（TIAM 色・フォント）に準拠
