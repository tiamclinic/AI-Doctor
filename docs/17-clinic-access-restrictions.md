## 概要

要件 §1.5 に基づき、**クリニックに来院した方を主対象**とする方針を、技術的にも補強する。URL を知っているだけでは診断フローに入れない／院発行の一回性トークンが必要、等のいずれか（または組み合わせ）を設計する。

## ゴール / 受け入れ基準

- [ ] 方式の ADR（例: Firebase Auth + Custom Claims / スタッフ発行ワンタイム URL / 院内 Wi‑Fi + IP 制限は非推奨理由も記載）
- [ ] 未認証時のランディング／誘導画面（来院案内 QR の文言と整合）
- [ ] 診断 API（`/api/diagnose` 等）への**ボット乱用対策**との関係を整理
- [ ] プライバシーポリシー・利用規約に「利用対象」を追記（リーガル連携）
