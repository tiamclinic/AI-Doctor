## 概要

MVP リリース前の品質保証・法令チェック・コピー磨き込み・ベータ公開を行う。受け入れ基準（要件定義書 §11）が全て満たされていることを最終確認する。

## ゴール / 受け入れ基準

- [ ] 受け入れ基準（要件定義書 §11）の全項目を満たす
- [ ] 利用規約・プライバシーポリシーがリーガルチェック済みで公開されている
- [ ] 薬機法・景表法に抵触する表現が UI / 診断文に含まれていない
- [ ] iOS Safari / Android Chrome / PC Chrome で全機能が動作する
- [ ] Firebase Analytics / Crashlytics が動いている
- [ ] ベータユーザー 5〜10 名分のフィードバックが得られている

## 設計メモ

### QA チェックリスト（抜粋）

- 機能テスト
  - 写真アップロード（JPEG / PNG / HEIC / WebP / 10MB 超）
  - 顔検出失敗時の UX
  - 診断文の言い回しチェック（禁止語が出ない）
  - シェアカード生成・ダウンロード
  - X / LINE 共有
  - AI 理想顔生成（同意あり / なし）
- 非機能テスト
  - 初期表示 LCP（Lighthouse）
  - スマホ実機での 60 秒以内完走
  - キーボード操作 / コントラスト比
  - OGP 表示確認

### リーガル

- 利用規約・プライバシーポリシーのドラフト → 弁護士レビュー（必要に応じて）
- 写真の取り扱い・OpenAI 送信に関する明記
- 「治る」「治療」「医療効果」「最も」「No.1」など禁止表現の最終チェック
- 「あくまで参考イメージ」表記の徹底（AI 理想顔）

### モニタリング

- Firebase Analytics: 主要イベント（`upload`, `face_detected`, `diagnosis_completed`, `share_clicked`, `portrait_generated`）
- Crashlytics: エラートラッキング
- OpenAI: コスト / 使用量

### ベータ公開

- 関係者 + フォロワー数名に共有
- フィードバックフォーム（Google Form でも可）
