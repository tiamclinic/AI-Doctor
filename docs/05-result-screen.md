## 設計メモ

### 配置

```
app/result/[id]/
  page.tsx                  結果画面
components/
  FaceLandmarkOverlay.tsx   Canvas で顔オーバーレイ
  ScoreCircle.tsx           円形プログレス（カウントアップ）
  ScoreRadar.tsx            6 指標レーダー（recharts 等）
  DiagnosisText.tsx         診断文レイアウト
  IdealPortrait.tsx         AI 理想顔（スケルトン → 画像）
```

### `[id]` の決定方法

- MVP は永続化しないので、結果は store に保持し `id` はクライアントで生成（nanoid）
- `id` を URL に乗せておくとリロード復元と SNS シェア URL の生成に便利
- Phase 2 で Firestore に保存して URL リンクで他人と共有できるように拡張

### 表示要素の順序（縦スクロール）

1. ヒーロー: 顔写真 + ランドマークオーバーレイ + 総合スコア大表示
2. 6 大指標レーダー
3. 各指標の数値リスト（バー表示）
4. 診断文（総評 → 強み → 注意点 → 推奨ケア → TIAM メッセージ）
5. AI 理想顔（あなたの黄金比イメージ）
6. シェアカード DL / SNS シェア（T-06 / T-08）

### Canvas オーバーレイ

- 写真を `<img>` ではなく `<canvas>` に描画し、その上にライン・点を重ねる
- 描画は `requestAnimationFrame` で初回 1 回のみ
- ランドマーク表示は ON/OFF 切替

## 実装メモ（完了時）

- 結果 ID は `nanoid(10)` でクライアント生成し、Zustand ストア (`resultId`) と URL の `[id]` を一致させて整合チェック。
- ストア未設定 or `id` 不一致時は `router.replace("/")` でトップへ戻す（MVP のため永続化なし）。
- `ScoreCircle` は SVG 円弧 + `requestAnimationFrame` のイージングで 0 → 値へカウントアップ（約 1.1 秒）。
- `ScoreRadar` は外部チャートライブラリを使わず独自 SVG。同心六角形（25/50/75/100）+ TIAM ゴールドの塗りつぶし。
